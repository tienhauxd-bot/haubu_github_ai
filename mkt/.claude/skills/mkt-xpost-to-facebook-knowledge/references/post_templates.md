# Post Templates — X Post to Facebook

Các mẫu bài viết Facebook được tối ưu cho input từ X posts. Mỗi template có cấu trúc rõ ràng, chỉ cần fill content vào.

---

## Template 1: Actionable Post (Bài hành động)

**Khi nào dùng:** X post giới thiệu tool, workflow, hoặc cách làm mới mà người đọc có thể áp dụng ngay.

**Cấu trúc:**

```
[HOOK - 2 dòng gây shock/tò mò, tập trung vào KẾT QUẢ người dùng đạt được]

[LINE BREAK]

[BỐI CẢNH - Pain point phổ biến mà tool/method này giải quyết. 2-3 dòng.]

[LINE BREAK]

[NHƯNG - Xung đột: tại sao cách cũ không hiệu quả]

[LINE BREAK]

[VÌ VẬY - Giới thiệu tool/method + giải thích bằng USER BENEFIT]
- Điểm 1: [Lợi ích cụ thể cho người dùng]
- Điểm 2: [Tiết kiệm bao nhiêu thời gian / công sức]
- Điểm 3: [Kết quả đo được]

[LINE BREAK]

[INSIGHT CÁ NHÂN - "Mình thấy hay ở chỗ..." hoặc "Bài học ở đây là..."]

[LINE BREAK]

[LAST DAB - Câu chốt hạ đáng nhớ]

[CTA - "Link ở comments nhé các bạn" hoặc CTA phù hợp]
```

**Ví dụ hook cho tool post:**
- "Một anh dev vừa build tool mà 1 người dùng nó thay cả team HR. 740 job listings, scan tự động, tạo CV riêng cho từng vị trí."
- "Mình vừa test thử tool này — 3 giờ làm thủ công giờ còn 5 phút."

## Template 2: Deep-dive Post (Bài phân tích sâu)

**Khi nào dùng:** X post chứa insight sâu, trend quan trọng, hoặc case study đáng phân tích kỹ.

**Cấu trúc:**

```
[HOOK - Tuyên bố bold / counter-intuitive]

[LINE BREAK]

[CÂU CHUYỆN - Kể ngắn bối cảnh: "Mình đọc được..." hoặc "Hôm qua lướt X thấy..."]
3-4 dòng dẫn dắt

[LINE BREAK]

[NHƯNG - Điều ít người nhận ra / góc nhìn sâu hơn]

[LINE BREAK]

[PHÂN TÍCH - Tại sao điều này quan trọng cho người dùng AI]
- Hệ quả cho công việc hàng ngày
- Cơ hội nếu nắm bắt sớm
- Cách áp dụng cho non-developer

[LINE BREAK]

[FRAMEWORK / CÁCH NGHĨ MỚI - Đúc kết thành nguyên tắc đơn giản]

[LINE BREAK]

[LAST DAB]

[CTA]
```

**Ví dụ hook cho trend/insight:**
- "AI vừa thay đổi cuộc chơi tìm việc. Không phải vì AI thông minh hơn — mà vì nó không biết mệt."
- "Có một thứ đang xảy ra trong thế giới AI mà 90% người dùng không nhận ra."

## Template 3: Listicle Post (Bài liệt kê)

**Khi nào dùng:** X post đề cập nhiều tools, features, hoặc updates cùng lúc — cần tách thành danh sách dễ scan.

**Cấu trúc:**

```
[HOOK - Số + promise cụ thể]

[LINE BREAK]

[CONTEXT - 1-2 dòng tại sao danh sách này đáng đọc]

[LINE BREAK]

1. [Item 1 — tên + 1-2 dòng giải thích USER BENEFIT]

2. [Item 2 — tên + 1-2 dòng giải thích USER BENEFIT]

3. [Item 3 — tên + 1-2 dòng giải thích USER BENEFIT]

(Tối đa 5-7 items)

[LINE BREAK]

[INSIGHT CÁ NHÂN - Cái nào mình thấy hay nhất và tại sao]

[LINE BREAK]

[LAST DAB + CTA]
```

## Template 4: Comparison Post (Bài so sánh)

**Khi nào dùng:** X post so sánh tools, approaches, hoặc có debate về A vs B.

**Cấu trúc:**

```
[HOOK - Câu hỏi gây tranh luận hoặc tuyên bố bold]

[LINE BREAK]

[BỐI CẢNH - Tại sao so sánh này quan trọng cho người dùng]

[LINE BREAK]

[Option A]:
- Phù hợp khi: [tình huống cụ thể cho USER]
- Kết quả: [output đo được]

[Option B]:
- Phù hợp khi: [tình huống cụ thể cho USER]
- Kết quả: [output đo được]

[LINE BREAK]

[KẾT LUẬN CÁ NHÂN - "Mình chọn X vì..." — phải có lập trường rõ ràng]

[LINE BREAK]

[LAST DAB + CTA]
```

## Quy tắc chung cho TẤT CẢ templates

### Hook
- 2 dòng đầu PHẢI gây tò mò hoặc shock
- Tập trung vào USER BENEFIT hoặc con số gây bất ngờ
- KHÔNG bắt đầu bằng "Hôm nay mình chia sẻ..."
- KHÔNG đề cập X.com hay tweet trong hook

### Thân bài
- Line breaks rõ ràng — dễ đọc trên mobile
- Mix câu ngắn + trung bình + dài tạo nhịp điệu
- Luôn có ví dụ cụ thể, số liệu khi có
- Translate mọi thứ technical thành user benefit

### Kết bài
- Last Dab: 1 câu đúc kết mạnh mẽ, dẫn ngược về hook
- CTA phù hợp ngữ cảnh:
  - Share link/tool: "Link ở comments nhé các bạn" hoặc "Comments mình gửi link nhé"
  - Tương tác: "Bạn đã thử dùng AI cho [việc X] chưa? Comment cho mình biết"
  - Save: "Save lại để dùng khi cần nhé"
